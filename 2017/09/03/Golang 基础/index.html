基本类型

数值类型

整数类型

  • int8, int16, int32, int64
  • uint8, uint16, uint32, uint64

浮点类型

  • float32
  • float64 默认

字符

布尔类型

var isActive bool  //默认false

上述类型为原始类型，不可改变，函数调用时传递的是这些值的副本

引用类型

包括slice，map，interface，chan，函数类型

引用类型的变量保存的是一个指针，指向底层数据结构，函数调用时传递的是这个指针的副本，而底层数据结构并没有传递。

本质上，我们可以理解函数的传递都是值传递，只不过引用类型传递的是一个指向底层数据的指针，所以我们在操作的时候，可以修改共享的底层数据的值，进而影响到所有引用到这个共享底层数据的变量。

slice

既然是引用类型，slice的指向是底层array。相当于包含3个字段的数据结构，一个指向数组的指针，一个slice长度，一个slice容量。

声明

aslice := make([]int, 4, 8)  //长度4，容量8指的是底层array的长度
bslice := []int{1,2,3,4} //没有像声明array一样指定长度，slice推断出长度为4，且容量也为4

比较：
var nilSlice []int // 指向array的指针为nil，还未初始化
cslice := []int{} // 指针肯定不为nil，代表一个空集合

slice 通过append添加内容，动态扩充底层array的容量。

从同一个array引出的slice，其中一个slice改变了某个值，则其他slice以及array中的改值都会改变。

函数调用时传递的是slice副本，即3个字段的数据结构。对其修改后，所指向的底层数组也会随之改变。

Map

dict := make(map[string]int) // 声明一个map，可以添加key:value使用了。
dict := map[string]int{} //一个空map

var dict map[string]int // nil, 并没有指向任何内存地址，需要开辟一块内存才能使用，如下。
dict = make(map[string]int)  或 dict = map[string]int{}

函数调用传递的同样是map的引用副本。

make 与 new

make只能用于内建类型（map、slice 和channel）的内存分配。new用于各种类型的内存分配。

new(T)分配了零值填充的T类型的内存空间，并且返回其地址，即一个*T类型的值。这个指针指向的内容的值为零。

make返回一个有初始值(非零)的T类型，而不是*T。

结构类型

函数调用过程，结构体传递的是其本身以及里面的值的副本。

通过传递结构体的指针, 可以达到我们的预期修改age的目的。

但是函数传递结构体时，如果结构体中含有引用类型，如Map，是会产生修改的。

函数方法

定义一个方法需要指定一个接收者。接收者分：指针接收者和值接收者。

使用值类型接收者定义的方法，在调用的时候，使用的其实是值接收者的一个副本，所以对该值的任何操作，不会影响原来的类型变量。

指针接收者传递的是一个指向原值指针的副本，指针的副本，指向的还是原来类型的值，所以修改时，同时也会影响原来类型变量的值。

调用指针接收者方法时，使用值或者指针都可以，Go的编译器自动会帮我们取指针。同样调用值接受者方法时，也可以使用指针，Go编译器自动会解引用。

在调用方法的时候，传递的接收者本质上都是副本，只不过一个是这个值副本，一是指向这个值指针的副本。指针具有指向原有值的特性，所以修改了指针指向的值，也就修改了原有的值。我们可以简单的理解为值接收者使用的是值的副本来调用方法，而指针接收者使用实际的值来调用方法。

接口

实现一个接口的方法，可以使用指针接收者和值接收者。二者区别为：实体类型以值接收者实现接口的时候，不管是实体类型的值，还是实体类型值的指针，都实现了该接口。实体类型以指针接收者实现接口的时候，只有指向这个类型的指针才被认为实现了该接口。
